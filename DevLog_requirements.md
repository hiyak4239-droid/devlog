# DevLog - 要件定義書

## 1. プロジェクト概要

- **アプリ名:** DevLog
- **目的:** プログラミング学習内容を記事として記録・公開
- **参考:** Qiita (https://qiita.com/)
- **技術スタック:**
  - Ruby on Rails 8.1.1
  - データベース: SQLite (開発) → PostgreSQL (本番想定)
- **対象ユーザー:**
  - 記事作成: 自分のみ
  - 記事閲覧: 第三者も可能
  - コメント: 第三者も可能（将来実装）

---

## 2. MVP機能（フェーズ1）

### 記事のCRUD操作

- **作成 (Create):** 新しい記事を作成
- **表示 (Read):** 記事の詳細を表示
- **編集 (Update):** 既存の記事を編集
- **削除 (Delete):** 記事を削除

### 制限事項（MVP）

- ユーザー認証なし（自分だけが使う前提）
- 公開機能なし（ローカル環境でのみ動作）

---

## 3. データ設計

### 記事テーブル (articles)

| 項目名 | カラム名 | データ型 | 必須 | 説明 |
|--------|----------|----------|------|------|
| タイトル | title | string | ✓ | 記事のタイトル |
| 本文 | content | text | ✓ | Markdown形式の本文 |
| 要約/概要 | summary | text | - | 記事の短い説明文 |
| カテゴリ | category | string | - | 1つ選択（例: フロントエンド、バックエンド）|
| タグ | tags | string (配列) | - | 複数可（例: Ruby, Rails, JavaScript）|
| 公開状態 | published | boolean | ✓ | true: 公開 / false: 下書き (デフォルト: false) |
| 作成日時 | created_at | datetime | ✓ | 自動設定 |
| 更新日時 | updated_at | datetime | ✓ | 自動更新 |

---

## 4. 画面一覧

### MVPで必要な画面

1. **記事一覧ページ (`/articles`)**
   - 全記事のリスト表示
   - タイトル、要約、カテゴリ、作成日時を表示
   - 「新規作成」ボタン

2. **記事詳細ページ (`/articles/:id`)**
   - 記事の内容を表示（Markdownを変換して表示）
   - タイトル、本文、カテゴリ、タグ、作成/更新日時
   - 「編集」「削除」ボタン

3. **記事作成ページ (`/articles/new`)**
   - 新規記事入力フォーム
   - タイトル、本文、要約、カテゴリ、タグ、公開状態の入力欄

4. **記事編集ページ (`/articles/:id/edit`)**
   - 既存記事編集フォーム
   - 記事作成ページと同じフォーム

---

## 5. ユーザー操作フロー（MVP）

```
記事一覧ページ (/articles)
├─ 「新規作成」ボタン → 記事作成ページ (/articles/new)
│                      └─ 「保存」 → 記事詳細ページ (/articles/:id)
│
├─ 記事タイトルクリック → 記事詳細ページ (/articles/:id)
                         ├─ 「編集」ボタン → 記事編集ページ (/articles/:id/edit)
                         │                 └─ 「保存」 → 記事詳細ページ
                         │
                         └─ 「削除」ボタン → 確認ダイアログ → 記事一覧ページ
```

## 6. 技術仕様

### Markdown対応

- 本文はMarkdown記法で入力
- 表示時にHTMLに変換
- 使用gem: `redcarpet` または `kramdown`

### スタイリング

- 最小限のCSS（Railsデフォルト + 軽微な調整）
- フェーズ2以降でBootstrapやTailwind CSS導入を検討
